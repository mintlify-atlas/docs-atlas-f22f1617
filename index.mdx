Radash is a modern utility library built for TypeScript-first projects. It covers arrays, async operations, currying, objects, strings, numbers, and type checking — all with full TypeScript generics and zero runtime dependencies. Think of it as a leaner, more expressive replacement for Lodash.

## Why Radash?

Radash was built to solve real problems that Lodash doesn't handle well in modern codebases:

<CardGroup cols={2}>
  <Card title="Tree-shakeable" icon="leaf">
    Import only what you use. Every function is a named export — no bloated bundle.
  </Card>
  <Card title="TypeScript-first" icon="shield-check">
    Generic signatures throughout. No `any` surprises — your editor will know the types.
  </Card>
  <Card title="Zero dependencies" icon="cube">
    No transitive vulnerabilities. Radash ships only its own code.
  </Card>
  <Card title="Async-native" icon="bolt">
    Async map, parallel execution, retry with backoff, defer — async is a first-class citizen.
  </Card>
</CardGroup>

## Quick look

```typescript
import * as _ from 'radash'

const gods = [
  { name: 'Ra',   power: 'sun',       rank: 100, culture: 'egypt' },
  { name: 'Loki', power: 'tricks',    rank: 72,  culture: 'norse' },
  { name: 'Zeus', power: 'lightning', rank: 96,  culture: 'greek' }
]

_.max(gods, g => g.rank)                         // => { name: 'Ra', ... }
_.sum(gods, g => g.rank)                         // => 268
_.fork(gods, g => g.culture === 'norse')         // => [[loki], [ra, zeus]]
_.sort(gods, g => g.rank)                        // => [ra, zeus, loki]
_.objectify(gods, g => g.name.toLowerCase())     // => { ra, zeus, loki }

// Error-safe async
const [err, god] = await _.tryit(api.gods.findByName)('Ra')

// Parallel async with concurrency limit
const allGods = await _.parallel(3, gods, async g => api.gods.fetch(g.name))
```
